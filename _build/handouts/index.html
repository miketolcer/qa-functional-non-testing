Functional & Non-functional Testing

We’ll be starting with some functional testing.

For your consideration, this is Simple Flash Cards (https://devmountain.github.io/qa_flashcards/), a very basic website where you can make a list of flash cards, then test yourself with those cards.

Here are a few requirements for you to consider while writing and running your tests:

  • In the top right corner, user will be able to access their profile, where they can select from decks, add new decks, share them, etc.

  • All flash cards should be visible on the home page of the application.

  • There should be a “+” button in the corner to add additional flash cards.

  • Flash cards should consist of a front and a back.

  • By default, there is a “flip” icon in the top right corner of the card to make it flippable.

  • The “i” button in the top left will let you edit or delete the card.

  • Flash cards should also show the last time the user reviewed the flash card and its outcome.

In your Trello board, create an outline of what tests you’ll need to create in order to test this flash cards site as thoroughly as possible.

Now for some non-functional testing!

For our first non-functional testing activity, your task is to review this website, User Interface (https://userinyerface.com/).

Write out a list of issues you encounter while attempting to complete the user flow of this website, good luck!
